## 首次使用：配置你的知识库

### 第一步：获取你的 API 凭证

1. 打开 Get 笔记网站：https://www.biji.com/subject
2. 进入你想要使用的知识库
3. 点击顶部的「API 设置」按钮
4. 复制你的 **API Key** 和 **知识库 ID**

### 第二步：配置你的知识库

**方式 1：使用 Web 配置界面（推荐 ⭐ 最便捷）**

直接在对话中说：
```
打开 Get 笔记配置界面
```

然后：
1. 浏览器会自动打开配置页面
2. 在表单中填入知识库名称、API Key、Topic ID
3. 描述字段可以输入 `auto` 让系统自动生成，也可以手动填写
4. 勾选"设为默认知识库"（可选）
5. 点击"保存配置"即可
6. 关闭浏览器标签页，服务会自动停止

**方式 2：通过对话配置**

在对话框中这样说：

```
帮我配置 Get 笔记：
- 知识库名称：政经参考
- API Key: 你的_API_KEY
- 知识库 ID: 你的_ID
- 描述：包含宏观政策、行业分析、政府报告
- 设为默认
```

或者更简单：

```
配置我的 Get 笔记：政经参考，API Key 是 xxx，知识库 ID 是 xxx，设为默认
```

**✅ 配置成功后，你就可以开始使用了！这个配置只需要做一次。**

## 检索范围控制（进阶）

### 四种检索模式

| 模式 | 命令 | 说明 |
|------|------|------|
| **默认模式** | `--default` | 仅搜索默认知识库 |
| **精准模式** | `--kb "库名"` | 搜索指定的一个或多个库 |
| **广播模式** | `--auto` | 🌟 AI 自动匹配最相关的库 |
| **广域模式** | `--all` | 搜索所有已配置的知识库 |

### 示例对话

**默认库搜索：**
```
搜索 Get 笔记（默认库）：房地产政策
```

**指定多个库搜索：**
```
在「政经参考」和「投资参考」中搜索 AI 趋势
```

**智能路由（推荐）：**
```
自动搜索最相关的库：最新政策分析
```

**全库检索：**
```
在所有知识库中搜索：年度总结
```

**组合使用：**
```
全库搜索新问题：AI 代理架构
```

### 范围继承

追问时会自动延续上次的检索范围，除非你显式指定新范围：

```
# 首次搜索（指定技术笔记库）
在「技术笔记」中搜索：什么是微服务？

# 追问（自动在技术笔记库中继续）
它有什么优缺点？

# 切换范围（后续追问也在全库）
在所有库中搜索：其他领域如何应用？
```

## 知识库描述管理

**为什么重要？** 知识库的描述用于「广播模式」的智能路由。描述越准确，AI 越能正确选择相关的库。

### 添加描述（配置时）

**在交互式配置时：**
```
帮我配置 Get 笔记：
- 名称：技术笔记
- API Key: xxx
- 知识库 ID: xxx
- 描述：该库主要涵盖 Python 开发、AI 架构，核心关键词包括 LLM、提示词工程、自动化，适用于技术研发
```

**或使用 "auto" 自动生成：**
```
帮我配置 Get 笔记：
- 名称：技术笔记
- API Key: xxx
- 知识库 ID: xxx
- 描述：auto
```

### 更新描述

```
更新「技术笔记」的描述为：该库主要涵盖 Python 开发、AI 代理，核心关键词包括 Claude 集成、自动化脚本、系统设计，适用于开发工作流优化
```

**或使用自动生成：**
```
自动生成「技术笔记」的描述
```

### 自动生成描述

让 AI 自动分析知识库内容并生成描述，最简单的方式：

**方式 1：Web 配置界面（推荐 ⭐ 最便捷）**
- 打开配置页面（对话中说：打开 Get 笔记配置界面）
- 在"描述"字段输入 `auto`
- 点击"保存配置"
- 系统会自动分析知识库内容并生成全面的描述
- 完成后会显示 ✅ 标记，描述字段会更新为生成的内容

**方式 2：命令行工具**
```bash
# 为单个知识库生成描述
python3 scripts/sync_metadata.py --kb "技术笔记"

# 为所有知识库生成描述
python3 scripts/sync_metadata.py --all
```

**格式示例：**
```
该库主要涵盖 Python 开发、AI 代理、系统设计，核心关键词包括 Claude 集成、自动化脚本、工作流优化，适用于开发工作流提升。
```

### 查看所有描述

```
显示所有知识库的描述
```

### 描述建议

- **格式**：`该库主要涵盖 [领域]，核心关键词包括 [标签1、标签2...]，适用于 [场景]。`
- **关键词**：包含核心领域关键词（如：Python、AI、投资）和主题标签（如：政策分析、技术架构）
- **长度**：简洁明了，控制在 150 字以内
- **更新频率**：定期更新（每月一次）以反映内容变化

## 常见问题

### Q1: 我需要每次都提供 API Key 吗？

**不需要！** 你只需要在首次配置时提供一次，之后 Claude 会自动记住。

### Q2: 追问时需要重复之前的问题吗？

**不需要！** Claude 会自动记住同一会话中的所有问答历史。

### Q3: 如何知道什么时候会创建新文件？

**以下情况会创建新文件：**
- 你明确说「开始新会话」或「新话题」
- 你切换到不同的知识库
- 你重启了 Claude Code

**否则会追加到同一个文件中。**

### Q4: 生成的文件保存在哪里？

**当前工作目录**。每次搜索会生成两个文件：
- `get_知识库名_时间.md` - 问答记录
- `get_知识库名_时间_引用.md` - 引用来源记录

### Q5: 一个 API Key 能用于多个知识库吗？

**可以！** 一个 API Key 可以访问你账户下的所有知识库，只需要提供不同的知识库 ID。

### Q6: 如何删除某个知识库配置？

**你说：**
```
删除「学习笔记」知识库配置
```

### Q7: 如何清空会话历史？

**你说：**
```
查看所有会话
```

然后：
```
清空会话 xxx（会话ID）
```

### Q8: 广播模式和精准模式有什么区别？

- **精准模式**：你明确指定搜索哪个库，适合知道信息在哪里的情况
- **广播模式**：AI 根据问题自动选择最相关的库，适合不确定信息在哪里的情况

### Q9: 如何让广播模式更准确？

为每个知识库添加详细的描述。描述应包含：
- 核心领域关键词
- 主要内容类型
- 适用场景
